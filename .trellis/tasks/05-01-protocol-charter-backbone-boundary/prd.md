# charter: runtime backbone 边界与术语

## Goal

定义 Runtime Backbone 的术语体系、语义边界和治理准则，作为后续映射、契约和验证的共同基础。

## Requirements

* 明确 Backbone / Mapping / Facade 的边界定义。
* 固化 thread/turn/item 在平台内的语义解释。
* 固化 Envelope v0 最小治理位与透传规则。
* 明确 ADR 模板与变更准入标准。

## Acceptance Criteria

* [ ] 形成术语表（包含核心实体、生命周期、错误类目）。
* [ ] 形成边界文档（Backbone vs Mapping vs Facade）。
* [ ] 形成 Envelope v0 字段规范与示例。
* [ ] 形成 ADR 模板并可用于后续子任务。

## Out of Scope

* 不落地具体 connector 代码实现。
* 不处理多 profile 并行治理。
