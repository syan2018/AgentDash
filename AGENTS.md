# AgentDashboard - Agent 协作说明

## 重要声明：用户主导设计原则

**本项目由用户完全主导设计和编写流程。** Agent 的角色是协助整理、记录和执行用户的意图，而非替用户做设计决策。

- 所有设计思路、架构决策、功能定义必须由用户明确提出
- Agent 负责将用户的想法转化为文档，提出建议但不越俎代庖
- 任何超出用户明确指示的"设计"内容都应被拒绝

## 项目背景

本项目旨在构建一个用于维护任意多个项目（包括项目中涉及的生产Task及其进度）的管理工程。

**核心灵感来源：**
1. 胡渊鸣的实践（多 Claude Code 实例并行工作）
2. vibe-kanban（管理多个 coding agent 的看板系统）
3. Trellis（AI 框架和工具包）

**用户的目标：** 用一个统一的看板控制多设备、多类型项目中 agent 协同运转，支持任意数字生产 SOP 的维护和管理。

## 协作模式

1. **用户提出意图** → **Agent 协助整理成文档**
2. **用户确认方案** → **Agent 执行具体操作**
3. **用户主导演进** → **Agent 记录变更历史**

## 工作法则

1. **小步快跑**：每次只处理少量内容
2. **文档精简**：保持可维护性，拒绝冗长
3. **充分讨论**：用户确认后再沉淀为文档
